PsTas Full Wiring Guide (Raspberry Pi Pico)

This wires a Raspberry Pi Pico to a DualShock 4 through PC817C optocouplers. Driving a Pico pin HIGH turns its optocoupler on, which presses the button. Driving it LOW releases the button. The Pico uses 3.3V logic, the same as the TinyS2, so the 220 ohm resistors are unchanged.

Flashing the Pico (much simpler than the TinyS2)

  1. Download the MicroPython UF2 file for your exact board from micropython.org/download (RPI_PICO for the original Pico, RPI_PICO_W for the Pico W, RPI_PICO2 for the Pico 2). Pick the one that matches your board.
  2. Hold the BOOTSEL button while plugging the Pico into USB. It mounts as a USB drive named RPI-RP2.
  3. Drag the UF2 file onto that drive. The Pico reboots running MicroPython.
  4. In Thonny, set the interpreter to MicroPython (Raspberry Pi Pico) and pick the port.
  5. Open controller/firmware/main.py, then File -> Save as -> MicroPython device, and save it as main.py.

The two ground rails (read this first)

There are two separate "ground" lines, and they must NEVER touch each other. Keeping them apart is the entire reason we use optocouplers.

  • Pico GND rail — the Pico's own GND. Every PC817 pin 2 (cathode) and every indicator LED's short leg connect here.
  • DS4 common rail — the controller's own button common. Every PC817 pin 3 (emitter) connects here.
Do NOT connect the Pico GND rail to the DS4 common rail. They stay fully separate. If you join them, you defeat the isolation and can confuse or damage the controller.

PC817C pinout

The notch or dot on the chip marks pin 1. With the notch up, pins go 1-2 down the left and 4-3 down the right.

        +--U--+
  pin 1 |o    | pin 4      pin 1 = anode    (input LED +, from the resistor)
  (A)   |     | (C)        pin 2 = cathode  (input LED -, to Pico GND rail)
        |     |            pin 3 = emitter  (output, to DS4 common rail)
  pin 2 |     | pin 3      pin 4 = collector(output, to the DS4 button pad)
  (K)   +-----+ (E)

Bench test one optocoupler (no multimeter needed)

Before wiring to the DS4, you can prove an optocoupler works using just an LED. This temporarily lights the LED from the output side, so you watch the optocoupler switch it on and off as you toggle the Pico pin.

   INPUT (Pico drives the opto):
     GP pin ---[220 ohm]---> opto pin1
     opto pin2 ---> Pico GND

   OUTPUT (opto switches the LED, test rig only):
     3V3 ---[220 ohm]---> LED long leg ---> LED short leg ---> opto pin4
     opto pin3 ---> Pico GND

In the Thonny shell, x.value(1) should light the LED and x.value(0) should turn it off. If the LED follows your commands, the optocoupler works end to end.

This output-side LED and 3V3 are a TEST RIG ONLY. Remove them before wiring the DS4. Pico 3V3 must never touch the DS4 side, or you break the isolation.

Find the DS4 common rail with a multimeter before soldering. In continuity mode, with the controller off, put one probe on the battery negative terminal and touch the other to each button pad. The pad that beeps is the ground side, which is the common rail. The collector (pin 4) goes to the signal pad; the emitter (pin 3) goes to the common rail.

Optional indicator LEDs

For each button, add a second branch off the same GP pin. This is fully independent of the optocoupler and only gives you a light.

   GP pin ---[220 ohm]--->|---- LED long leg (anode)
                                LED short leg (cathode) ----> Pico GND rail

Use the same 220 ohm value. The long leg goes toward the resistor; the short leg goes to the Pico GND rail. If it does not light, flip it first.

Give the indicator LED its OWN 220 ohm resistor, on its own breadboard column. Do NOT let it share the optocoupler's resistor or sit on the same node after a shared resistor. The optocoupler's internal LED turns on at about 1.2V, which clamps a shared node below the 2V the indicator LED needs, so the indicator would never light. Two separate resistors, two separate columns, both jumpered to the same GP pin, keeps both happy.

Power and isolation

The Pico is powered over its own micro-USB cable. The PC817 output side (pins 3 and 4) is powered only by the DS4. The two sides share no wires. Leave the DS4 powered normally over its own USB; we only tap the button contacts.
